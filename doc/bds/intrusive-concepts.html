Intrusive Concepts¶

template<typename C>
concept bds::Stateless¶

A type is stateless if it is both empty and trivially constructible. Together, these properties ensure that there is no need to store a reference to a specific instance of a stateless class, because constructing a new instance on the fly has no cost and is always equivalent:

  • Both instances have the same data, because they are both empty
  • The act of constructing the new instance is side-effect-free because the constructor is trivial (and is therefore a no-op that will be optimized away)

See also

The documentation for bds::compressed_invocable_ref contains an explanation of how this is used in BDS.

template<typename T>
concept bds::CompressedSize¶

This concept is satisfied when the type is Integral type or when it is the special empty struct bds::no_size. The intention is to allow containers to either store their size inline (so that std::size will be \(O(1)\)) or to save space and compute it on the fly (but then std::size will be \(O(n)\)). This is usually done by accepting the size type as a template parameter, which is constrained to to be a CompressedSize. The size member variable itself is usually declared with [[no_unique_address]], so that it does not take any space when it is bds::no_size.
